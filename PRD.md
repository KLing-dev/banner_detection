# 横幅检测与违规内容识别告警模块 详细需求规格说明书
## 一、项目概述
开发一款基于 YOLO12（目标检测）+ByteTrack（目标追踪）+PaddleOCR（文字识别）的横幅检测与违规内容识别告警模块，运行于 Python 3.11 环境，适配 CUDA 11.8/12.6、PyTorch 2.6.0 技术栈。模块支持视频文件/模拟 RTSP 视频流输入，实现「横幅检测→文字提取→违规词比对→告警输出」全流程，分 4 个阶段迭代开发可独立运行的 Demo，最终输出标注检测结果的视频及标准化违规告警信息。

## 二、通用环境要求
| 依赖项         | 版本/规格                          | 备注                     |
|----------------|------------------------------------|--------------------------|
| Python         | 3.11.x                             | 禁止低于 3.11.0          |
| CUDA           | 11.8 或 12.6                       | 二选一，需与 PyTorch 兼容 |
| PyTorch        | 2.6.0                              | 需匹配 CUDA 版本         |
| YOLO12         | 官方 ultralytics 实现版本          | 基于 YOLOv8 架构迭代版   |
| ByteTrack      | 适配 Python 3.11 的最新稳定版      | 目标追踪核心依赖         |
| PaddleOCR      | 2.7+（支持 CUDA 加速）             | 中英文通用 OCR 模型      |
| 辅助工具       | rtsp-simple-server（模拟 RTSP 流） | 本地视频转 RTSP 流       |

## 三、分阶段 Demo 详细需求
### 阶段 1：YOLO12 横幅检测权重训练 Demo
#### 1. 核心目标
训练适用于横幅场景的 YOLO12 自定义权重，具备对视频帧中横幅目标的精准检测能力。
#### 2. 输入
| 输入项                | 格式/规格要求                                                                 |
|-----------------------|-------------------------------------------------------------------------------|
| 横幅标注数据集        | 文件夹结构：<br>- dataset/images/train/（训练集图像，MP4/PNG/JPG）<br>- dataset/images/val/（验证集图像）<br>- dataset/labels/train/（训练集标注，YOLO 格式 TXT）<br>- dataset/labels/val/（验证集标注）<br>标注规则：单类别（class_id=0，类别名=banner），标注框覆盖完整横幅 |
| YOLO12 配置文件       | YAML 格式，包含：<br>- nc: 1（类别数）<br>- names: ['banner']（类别名）<br>- 训练超参数（可选） |

#### 3. 输出
| 输出项                | 格式/规格要求                                                                 |
|-----------------------|-------------------------------------------------------------------------------|
| 训练权重文件          | .pt 格式，包含 best.pt（验证集最优权重）、last.pt（最后一轮权重）             |
| 训练日志              | CSV 格式 + 可视化图表（PNG），包含：损失曲线、mAP 曲线、学习率变化曲线        |
| 权重评估报告          | TXT/JSON 格式，包含：Precision（精度）、Recall（召回率）、mAP50、mAP50-95      |

#### 4. 功能要求
1. 支持指定 CUDA 设备（如 `cuda:0`），适配 11.8/12.6 版本，自动检测 GPU 显存并调整 batch_size；
2. 训练超参数可配置（默认值）：<br>- epochs=100、batch_size=16（显存不足时自动降级为 8/4）<br>- imgsz=640、lr0=0.01、weight_decay=0.0005；
3. 训练过程实时打印损失值（box_loss、cls_loss），每 10 轮输出一次验证集指标；
4. 训练完成后自动生成权重评估报告，标注 val 集漏检/误检案例。

#### 5. 验收标准
1. 权重文件生成无报错，best.pt 大小 ≥50MB；
2. val 集 mAP50 ≥0.85，横幅检测准确率 ≥80%；
3. 训练日志完整，无中断/NaN 异常。

### 阶段 2：横幅检测与追踪 Demo（模拟 RTSP 输入）
#### 1. 核心目标
将本地视频转为 RTSP 流，基于阶段 1 权重实现横幅检测+ByteTrack 追踪，输出标注视频。

#### 2. 输入
| 输入项                | 格式/规格要求                                                                 |
|-----------------------|-------------------------------------------------------------------------------|
| YOLO12 权重文件       | 阶段 1 生成的 best.pt                                                         |
| 测试视频文件          | MP4/AVI 格式，分辨率 ≥1080P，时长 ≥30s                                        |
| RTSP 服务配置文件     | rtsp-simple-server 的 config.yml（默认端口 8554）                             |

#### 3. 输出
| 输出项                | 格式/规格要求                                                                 |
|-----------------------|-------------------------------------------------------------------------------|
| RTSP 流地址           | 格式：rtsp://localhost:8554/stream，可通过 VLC 验证访问                        |
| 标注检测视频          | MP4 格式，帧率与原视频一致，画面包含：横幅边界框（彩色）、追踪 ID、置信度       |
| 检测日志              | JSON 格式，每帧记录：frame_id、banner_ids、每个 banner 的（x1,y1,x2,y2,conf）|

#### 4. 功能要求
1. 视频转 RTSP 流：<br>- 启动 rtsp-simple-server 服务；<br>- 通过 ffmpeg 将本地视频推流至 RTSP 地址：<br>  `ffmpeg -re -i test.mp4 -c copy -f rtsp rtsp://localhost:8554/stream`；
2. 横幅检测：<br>- 置信度阈值 ≥0.5，NMS 阈值 ≥0.45，过滤小面积误检框（面积 < 1000 像素）；
3. ByteTrack 追踪：<br>- track_thresh=0.5、track_buffer=30、match_thresh=0.8；<br>- 同一横幅跨帧追踪 ID 稳定，无频繁跳变；
4. 视频标注：<br>- 不同追踪 ID 用不同颜色（如 ID1=红色，ID2=蓝色）；<br>- 边界框旁显示：ID:XXX, Conf:0.XX。

#### 5. 验收标准
1. RTSP 流可正常访问，无卡顿/花屏；
2. 检测视频无漏检明显横幅（≥90% 检出率），追踪 ID 连续 10 帧以上稳定；
3. 处理 1080P 视频帧率 ≥10FPS（GPU 加速）。

### 阶段 3：横幅文字识别 Demo
#### 1. 核心目标
对检测到的横幅 ROI 区域进行 OCR 识别，提取文字内容并标注到视频。

#### 2. 输入
| 输入项                | 格式/规格要求                                                                 |
|-----------------------|-------------------------------------------------------------------------------|
| 阶段 2 的 RTSP 流/检测视频 | -                                                                             |
| 阶段 2 的检测日志     | JSON 格式（含横幅边界框坐标）                                                |
| PaddleOCR 预训练模型  | 中英文通用模型（ch_PP-OCRv4_det_infer + ch_PP-OCRv4_rec_infer）                |

#### 3. 输出
| 输出项                | 格式/规格要求                                                                 |
|-----------------------|-------------------------------------------------------------------------------|
| 文字识别结果文件      | JSON 格式，包含：frame_id、banner_id、bbox、text、text_conf（文字置信度）       |
| 标注文字的视频文件    | MP4 格式，在阶段 2 标注基础上，横幅下方叠加识别文字+置信度                    |

#### 4. 功能要求
1. ROI 裁剪：根据横幅边界框坐标裁剪区域，边缘扩展 5% 避免文字截断；
2. 图像预处理：<br>- 灰度化、高斯降噪（核 3×3）、对比度增强（1.2 倍）；<br>- ROI 缩放至 ≥320×320，保证 OCR 识别精度；
3. OCR 识别：<br>- 启用 PaddleOCR CUDA 加速，过滤 text_conf < 0.5 的结果；<br>- 合并同一 banner 连续 5 帧的重复文字（相似度 ≥90%）；
4. 文字标注：<br>- 文字颜色与边界框一致，字体大小适配横幅尺寸；<br>- 置信度 <0.7 的文字标注为灰色，≥0.7 为白色。

#### 5. 验收标准
1. 10 个测试横幅的文字识别准确率 ≥85%；
2. 无文字截断/漏识别，重复文字合并无冗余；
3. 标注视频文字清晰，无遮挡/重叠。

### 阶段 4：违规词检测与告警 Demo（整体集成）
#### 1. 核心目标
集成全流程，实现违规词比对、实时告警、最终视频输出。

#### 2. 输入
| 输入项                | 格式/规格要求                                                                 |
|-----------------------|-------------------------------------------------------------------------------|
| 阶段 3 的文字识别结果 | JSON 格式                                                                     |
| 违规词库              | 支持两种方式：<br>1. 手动编写（代码内列表）<br>2. 外部上传（TXT/JSON，每行/每项一个违规词） |
| 阶段 3 的标注视频     | MP4 格式                                                                      |

#### 3. 输出
| 输出项                | 格式/规格要求                                                                 |
|-----------------------|-------------------------------------------------------------------------------|
| 最终检测视频          | MP4 格式，违规词红色高亮，非违规词白色，横幅边界框标红（违规时）                |
| 违规告警日志          | TXT/JSON 格式，包含：timestamp（视频时间戳，格式 HH:MM:SS.ms）、illegal_word、banner_id、text |
| 实时告警输出          | 控制台打印：[YYYY-MM-DD HH:MM:SS] 出现违规词：XXX                             |

#### 4. 功能要求
1. 违规词库管理：<br>- 支持 TXT（每行一个违规词）、JSON（{"illegal_words": ["词1","词2"]}）格式导入；<br>- 支持大小写不敏感匹配、包含匹配（如“违规词”在“XX违规词XX”中即触发）；
2. 时间戳转换：将 frame_id 转为视频时间戳（精度 ≤100ms）；
3. 告警输出：<br>- 实时控制台打印告警信息，含精确时间戳；<br>- 告警日志实时写入文件，按时间戳排序；<br>- 视频中违规词用红色加粗标注，横幅边界框改为红色闪烁；
4. 全流程集成：<br>- 支持一键运行：RTSP 流生成 → 横幅检测 → 文字识别 → 违规比对 → 告警+视频输出；<br>- 异常处理：OCR 识别失败/无文字时跳过比对，不触发告警。

#### 5. 验收标准
1. 违规词库导入无报错，包含匹配规则生效；
2. 含违规词的横幅触发告警，控制台/日志输出格式正确；
3. 最终视频违规词标注准确，边界框标红；
4. 全流程处理 5 分钟视频无崩溃，告警无漏报/误报。

## 四、Demo 代码结构要求
```

├── stage1_train/          # 阶段1：权重训练

├── stage3_ocr/            # 阶段3：OCR识别
│   ├── ocr_recognize.py   # OCR主脚本
│   └── output/            # 文字识别结果/标注视频

│   └── output/            # 最终视频/告警日志
└── requirements.txt       # 环境依赖清单
```

### 总结
1. 分 4 个阶段迭代开发，每个阶段输出独立可运行的 Demo，前一阶段为后一阶段的前置条件；
2. 核心技术栈：YOLO12（横幅检测）+ByteTrack（追踪）+PaddleOCR（文字识别），需适配指定的 Python/CUDA/Torch 版本；
3. 关键验收标准：权重 mAP50≥0.85、横幅检出率≥90%、文字识别准确率≥85%、违规告警无漏报/误报。
